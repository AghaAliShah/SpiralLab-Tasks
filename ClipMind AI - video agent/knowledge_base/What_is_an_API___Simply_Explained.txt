Hi everyone and welcome to Apna College. Aaj hum baat karne waale hain yaar APIs ki. API matlab Application Programming Interface. 

Ab hum Zomato, Uber, Ola jaisi apps ka use karte hain aur in saari apps ke andar hume ek map dikhai deta hai. Us map ke andar agar hum Zomato se order place karte hain to hume pata chal jata hai ki hamare delivery waale bhaiya kahan pahuche hain. 

To yaar Zomato ne upar ek satellite to laga nahi rakha jiski वजह se wo saari ki saari locations ko track kar pata hai. To Zomato map kahan se lekar aata hai? Zomato map laata hai Google Maps naam ki ek system se. Ab ye system humne ek app ke form mein use kar rakha hai, lekin Zomato directly Google Maps ke data ko access nahi kar sakta. Isiliye beech mein ek security ki layer Google Maps ne laga rakhi hai aur isi layer ko wo kehta hai Google Maps ki API. 

Ab Zomato ko agar data access karna hai to wo API se puchega, Google Maps se jaakar directly nahi puchega. To API ko hum ek messenger ki tarah samajh sakte hain. Wo messenger jo data ko ek jagah se dusri jagah lekar jaata hai. 

To Zomato ko agar Maps ka data chahiye to wo API ko request karega ki aap system se puchiye ki kya hume wo data de sakte hain ya nahi de sakte. API fir jaakar apne system yani Google Maps ke paas jaakar puchta hai unse ki kya mai data wapas Zomato ko de du? Agar system kehta hai haan, to jo data system API ko deta hai wahi API jaakar Zomato ko de deta hai. Agar system kehta hai nahi, to API kya karta hai? Ek error return kar deta hai Zomato ko, isiliye kabhi-kabhi network agar kharab hota hai to map hume pura ka pura display nahi ho pata na hi hum apne khaane waale bhaiya ko track kar pate hain. 

Ab API jo hai ek tarike se kisi bhi system ka 'How' nahi batata, 'What' batata hai. Matlab system ke paas agar burger banane ki recipe hai to API wo recipe nahi batayega kisi bhi app ya website ko, wo final tayaar burger lekar app ya website ko de dega and app ya website isi final product ke saath baad mein kaam kar rahe honge. 

Usi tarike se agar hum koi nayi application download karte hain to uske andar hamare paas options aate hain: 'Signup with Google', 'Signup with Facebook' jaise. Ab ye features kyu exist karte hain? Ab kisi ne bhi agar nayi app banai hai to usko jaakar ek-ek user ko verify na karna pade ki iske credentials sahi hain ya nahi, kyunki wo kaam already Google aur Facebook ke bade-bade databases ne kiya hua hai. To apps kya karti hain? Wo directly Google se puch leti hain Google API ke through ya Facebook API ke through ki kya aapke database mein ye ek valid user jo hai wo exist karta hai? Agar wo valid user exist karta hai to unko apni app mein entry de dete hain, nahi to us entry ko deny kar dete hain. 

To isme sawaal ye uthta hai ki fir to koi bhi app hamare data ka misuse kar sakti hai. To isme API ek alag tarah ki protection layer lekar aate hain kiske through? API keys ke through. API key ek security code hota hai jo koi bhi system apne developer ko deta hai jo bhi developer uske API ko access karne ki koshish kar raha hai. 

Ab kuch-kuch APIs jo hoti hain yaar wo free to use bhi hoti hain matlab unme koi API key nahi hota. Lekin mostly APIs ke andar API key hota hai. Kuch-kuch mein ye API key free hota hai, lekin kuch-kuch APIs jo invaluable data ko access provide kar rahi hain unke liye jo API key hoti hai wo paid key hoti hai. 

To API key ki help se hamare paas do fayde ho jaate hain ek system ke paas. Pehla fayda ye ki security hamare paas aa jaati hai. Security kis tarike se? Jaise Zomato directly Google Maps ke data ko access nahi kar pa raha tha, waise hi koi bhi app ya website directly kisi bhi system ke data ko access nahi kar pata. 

Uske saath-saath ek aur fayda hota hai hamara monitoring ka. Monitoring yani koi bhi API jo hai uspar unlimited requests aap nahi bhej sakte. Uske saath-saath jo bhi requests hain unko aapko sahi tarike se bhejnah padega, aap nonsensical requests nahi bhej sakte and saath ke saath aap illegal activities nahi kar sakte kyunki wo saari activities aapki track ho rahi hain, monitor ho rahi hain API keys ki help se.
